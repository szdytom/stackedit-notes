ARC122E Increasing LCMs题解

之后做这道题感觉很简单，就是不知道考试的时候自己能否想到。

考虑从后往前填 —— 首先考虑填限制最大的位置。

假设当前到第 xxx 位，还未使用的数集为 A′A'A′，那么，一个数字 ai∈A′a_i\in A'ai​∈A′ 能够被放到第 xxx 位上当且仅当

gcd⁡(lcm⁡(aj),ai)<ai(aj∈A′ and aj≠ai)\gcd(\operatorname{lcm}(a_j),a_i)< a_i(a_j\in A'\texttt{ and } a_j\neq a_i)gcd(lcm(aj​),ai​)<ai​(aj​∈A′ and aj​=ai​)

但是一个位置上可能出现多个满足条件的数，选择哪一个？任意取一个就好了！若有多个数同时满足，那么他们相互之间，在质因数分解下，一定相互都会存在某个自己的因数多于其他数字的该因数，即无论谁先放，都不能将剩下的数的因数覆盖完，又即无论谁先放，总会让最小公倍数出现增加的部分。

当然，如果没有这样的数，就报告无解。

于是直接 O(n3log⁡n)\mathcal O(n^3\log n)O(n3logn) 的暴力就解决了，一个实现细节是上面的 lcm⁡\operatorname{lcm}lcm 会爆 long long，但是我们可以找到等价形式：

(∗)⇔lcm⁡(gcd⁡(aj,ai))<ai(aj∈A′ and aj≠ai)(*)\Leftrightarrow\operatorname{lcm}(\gcd(a_j,a_i))< a_i(a_j\in A'\texttt{ and }a_j\not= a_i)(∗)⇔lcm(gcd(aj​,ai​))<ai​(aj​∈A′ and aj​=ai​)

做完了，挺暴力的。

参考实现：

#include <iostream>
#include <cstdio>
#include <cstring>
#include <algorithm>
using namespace std;

const int N = 105;
int n;
long long a[N];
 
long long gcd(long long a, long long b) { return b ? gcd(b, a % b) : a; }
long long lcm(long long a, long long b) { return a / gcd(a, b) * b; }
 
int main() {
    cin >> n;
    for (int i = 1; i <= n; ++ i) cin >> a[i];
 
    for (int i = n; i >= 1; -- i) {
        a[0] = a[i];
        for (int j = 0; j < i; ++ j) {
            long long s = 1;
            swap(a[i], a[j]);
            for (int k = 1; k < i; ++ k)
                s = lcm(s, gcd(a[k], a[i]));
            if (s != a[i]) break;
            if (j == i - 1) {
                cout << "No" << endl;
                return 0;
            }
        }
    }
    
    cout << "Yes" << endl;
    for (int i = 1; i <= n; ++ i) cout << a[i] << " ";
    cout << endl;
    return 0;
}
