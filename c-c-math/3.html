基本计数原理

如果你有过处理简单数数问题的经历，你可能早已听说过“排列”、“组合”这些词。如果你有更深入地研究数数的经历，你可能就会知道一些看似十分简单的问题，在求解的过程中也会遇到许多困难。这些困难一般有两种情况：

  1. 难以根据具体问题列出一般的数学表达式。
  2. 难以将数学表达式化简至一个可以接受的形式或求出其值。

解决这些困难也是组合数学研究的一大部分，下面指出的四个基本计数原理1将帮助解决第一步。

加法原理

加法原理是十分显然的，即“整体等于部分之和”。

例题1 一名学生要在数学类课程或者计算机类课程中选修一门课程。其中数学类课程有 333 门，计算机类课程有 444 门，那么他有多少种可能的选修方案？

这里隐含了一个条件：数学类的 333 门课程与计算机类的 444 门课程是相互独立的，即数学类的 333 门课程中没有一门是计算机类的，计算机类的 444 门课程中没有一门是数学类的。因此，总的选择方案有

3+4=73+4=73+4=7

这个简单的计数原理被称为加法原理（addition principle）：设集合 SSS 被划分为两两不相交的集合 S1,⋯ ,SnS_1,\cdots,S_nS1​,⋯,Sn​。则 SSS 包含的对象数目等于它的每个部分的对象数目之和，即

card⁡S=card⁡S1+⋯+card⁡Sn\operatorname{card}S=\operatorname{card}S_1+\cdots+\operatorname{card}S_ncardS=cardS1​+⋯+cardSn​

练习1 一名学生上学要么骑自行车去，要么走路去。其中骑自行车有 222 种路线，走路有 666 种路线，请问他一共有多少种路线上学？

乘法原理

第二个基本计数原理则稍微复杂一些，它的一个常见形式是：如果第一项任务有 ppp 中不同的结果，（无论第一项任务的结果如何）第二项任务有 qqq 中不同的结果，则连续执行两项任务一共有 p×qp\times qp×q 种结果。

这个想法形式化的说时，被称为乘法原理（multiplication principle）：若集合 S={(a,b)∣a∈U1,b∈U2}S=\{(a,b)\vert a\in U_1,b\in U_2\}S={(a,b)∣a∈U1​,b∈U2​}，即 SSS 是对象的有序对 (a,b)(a,b)(a,b) 的集合，其中第一个对象 aaa 来自于 U1U_1U1​，第二个对象 bbb 来自于 U2U_2U2​，则 SSS 的大小为 U1U_1U1​ 的大小乘 U2U_2U2​ 的大小，即

card⁡S=card⁡U1×card⁡U2\operatorname{card}S=\operatorname{card}U_1\times\operatorname{card}U_2cardS=cardU1​×cardU2​

对于多个集合时，上述原理也适用，即

card⁡{(e1,⋯ ,en)∣e1∈U1,⋯ ,en∈Un}=card⁡U1×⋯×card⁡Un\operatorname{card}\{(e_1,\cdots,e_n)\vert e_1\in U_1,\cdots,e_n\in U_n\}=\operatorname{card}U_1\times\cdots\times\operatorname{card}U_ncard{(e1​,⋯,en​)∣e1​∈U1​,⋯,en​∈Un​}=cardU1​×⋯×cardUn​

例题2 一名学生要在数学类课程和计算机类课程中各选修一门课程。其中数学类课程有 333 门，计算机类课程有 444 门，那么他有多少种可能的选修方案？

选修了哪门数学类课程并不影响选修哪门计算机类课程，因此总的可能的方案数为

3×4=123\times4=123×4=12

例题3 指出 332552516868003325525168680033255251686800 的正整数因数个数。

我们首先可以将这个数分解质因数

33255251686800=24×37×52×113×13433255251686800=2^4\times3^7\times5^2\times11^3\times13^433255251686800=24×37×52×113×134

根据算术基本定理（the fundamental theorem of arithmetic），每个满足条件的因子可以写成，

2a×3b×5c×11d×13e2^a\times3^b\times5^c\times11^d\times13^e2a×3b×5c×11d×13e

的形式，其中 0⩽a⩽4,0⩽b⩽7,0⩽c⩽2,0⩽d⩽3,0⩽e⩽40\leqslant a\leqslant4,0\leqslant b\leqslant 7,0\leqslant c\leqslant 2,0\leqslant d\leqslant 3,0\leqslant e\leqslant 40⩽a⩽4,0⩽b⩽7,0⩽c⩽2,0⩽d⩽3,0⩽e⩽4，由此可知， aaa 有 555 种选择，bbb 有 888 种选择，ccc 有 333 种选择，ddd 有 444 种选择，eee 有 555 选择，且这些选择相互之间是不相关的。故因子总数为

5×8×3×4×5=24005\times8\times3\times4\times5=24005×8×3×4×5=2400

减法原理

有时满足某些条件的情况数或许难以直接求出，但是不满足条件的情况数则相对容易求出时，我们可以考虑使用一个新的计数原理——减法原理（subtraction principle）：若集合 SSS 是集合 UUU 一个子集，设

S‾=U−S\overline{S}=U-SS=U−S

为 SSS 关于 UUU 的补集。则 SSS 中包含的元素个数由下面法则给出：

card⁡S=card⁡U−card⁡S‾\operatorname{card}S=\operatorname{card}U-\operatorname{card}\overline{S}cardS=cardU−cardS

例题4 某系统的密码为只可能包含字符 0,1,⋯ ,9,a,b,⋯ ,z0,1,\cdots,9,a,b,\cdots,z0,1,⋯,9,a,b,⋯,z 这 101010 个数字和 262626 个小写字母组成的长度为 666 的字符串。则该系统的所有可能的密码中有多少种包含重复字符？

我们考虑使用减法原理，该系统所有可能的密码数量为

366=217678233636^6=2176782336366=2176782336

而其中没有重复字符的密码数量为

36×35×34×33×32×31=140241024036\times35\times34\times33\times32\times31=140241024036×35×34×33×32×31=1402410240

因此，有重复的密码数量为

2176782336−1402410240=7743720962176782336-1402410240=7743720962176782336−1402410240=774372096

除法原理

最后一个基本的级数原理叫做除法原理（division principle）：若 SSS 为一个有限集合，把它划分为 kkk 个互不相交的部分使得每个部分包含的对象数目均为 qqq，则

card⁡S=kq\operatorname{card}S=kqcardS=kq

例题5 在书架上有 240240240 本书，其中每层有 606060 本书，求书架的层数。

运用除法原理，书架的层数为

24060=4\frac{240}{60}=460240​=4

排列与组合

例题6 有 666 个橙子和 999 个苹果，现在你需要用其中的一些组成一个果篮（至少包含一个水果），求组成果篮的方案数？（两个组成方案视为不同当且仅当其中的橙子数量或苹果数量不同）

这道例题很容易使用乘法原理和减法原理解决，即先忽略篮子非空的限制，随后再减去一种情况。

7×10−1=697\times10-1=697×10−1=69

我们之所以可以这么考虑，是基于题目中括号内的话“两个组成方案视为不同当且仅当其中的橙子数量或苹果数量不同”。这句话提示我们一个隐含的条件，即苹果和苹果是没有区别的，橙子和橙子也是没有区别的。因而果篮的方案与具体那个苹果或里橙子装入篮子无关，而只与每种水果的数量有关。如果我们在各个水果之间加以区别，果篮的方案数就会增大。

事实上，这个例子是具有指导性意义的。我们不仅要关注与计数对象本身，更要关注于如何区分两种方案。具体而言，我们一方面关注于同一类对象之间是否加以区分，如果加以区分，我们还要关注选出的对象是否有顺序。考虑到顺序的选择通常称为排列（permutation），而与顺序无关的选择则称为组合（combination）。我们首先研究排列的情况。

例题7 求从 101010 个人中选出 333 人排队的方案数。

我们运用乘法原理，排在一个位置的人有 101010 种选法，第二个位置还剩 999 种，第三个位置还剩 888 种，于是总的方案数为

10×9×8=72010\times9\times8=72010×9×8=720

在通常的情况下，从 nnn 个元素中有顺序地选出 rrr 个的方案数由

P(n,r)=n×(n−1)×⋯×(n−r+1)P(n,r)=n\times(n-1)\times\cdots\times(n-r+1)P(n,r)=n×(n−1)×⋯×(n−r+1)

给出。对于非负整数 nnn，我们定义 nnn 的阶乘（factorial，记作 n!n!n!）为 111 到 nnn 的乘积，即

n!=1×2×⋯×nn!=1\times2\times\cdots\times nn!=1×2×⋯×n

特别地，令 0!=10!=10!=1。于是上面的式子可以重写成

P(n,r)=n!(n−r)!P(n,r)=\frac{n!}{(n-r)!}P(n,r)=(n−r)!n!​

类似 n×(n−1)×⋯×(n−r+1)n\times(n-1)\times\cdots\times(n-r+1)n×(n−1)×⋯×(n−r+1) 这种部分阶乘的式子经常出现，考虑到它和乘方的相似性，我们称之为 nnn 的 rrr 次下降幂2（factorial power，记作 nr‾n^{\underline{r}}nr​） 。

例题8 指出使用 555 个字母各至多一次组成的长度为 444 的字符串的数量。

这个问题相当于有顺序的从 555 个对象中选择 444 个的排列数，即

54‾=1205^{\underline{4}}=12054​=120

例题9 求从 101010 个人中选出 333 人的方案数。

我们在例题7已经知道，101010 个人选出 333 个人的排列数为 720720720。我们考虑到每个排列都恰好由依次执行下面两个任务的结果产生：

  1. 从 101010 个人中选出 333 个人。
  2. 指定 333 个人的顺序。

我们知道无论步骤1的结果如何，步骤2始终有 3!=63!=63!=6 种方案，且连续执行的总方案数为 720720720，根据除法原理，我们得到第一步的方案数为

7206=120\frac{720}{6}=1206720​=120

这个例子的方法具有一般的通用意义。我们把从 nnn 个对象中选出 rrr 个的方案数称为组合数（binomial coefficient，又名二项式系数，记作 (nr)\binom nr(rn​)），其计算公式由

(nr)=n!r!(n−r)!\binom nr=\frac{n!}{r!(n-r)!}(rn​)=r!(n−r)!n!​

给出，其中 0⩽r⩽n0\leqslant r\leqslant n0⩽r⩽n。

例题10 用 333 个字母 a、444 个字母 b 和 222 个字母 c 一共可以组成多少个不同的字符串？

我们一共有 3+4+2=93+4+2=93+4+2=9 个字母，因此我们最终得到的字符长度为 999。在这 999 个位置中，我们首先选出 333 个位置放置字母 a，其方案数由

(93)=84\binom 93=84(39​)=84

给出。现在剩下了 9−3=69-3=69−3=6 个空位，我们在其中选出 444 个位置放置字母 b，其方案数由

(64)=15\binom 64=15(46​)=15

给出。最后的剩下的 222 个空位只能全部放置字母 c。根据乘法原理，总的方案数为

84×15=126084\times15=126084×15=1260
